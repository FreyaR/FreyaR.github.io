Java继承

Posted on 2019-11-17 | In Java

类、超类和子类

  • 超类：已存在的类，也被称为基类或父类
  • 子类：新类，也被称为派生类或孩子类

Java中每个类都只能有一个直接父类，可以有无限多个间接父类

调用超类的方法应使用super关键词

在子类中可以增加域、增加方法或覆盖超类的方法，然而绝对不能删除继承的任何域和方法

通过super实现对超类构造器的调用，使用super调用构造器的语句必须是子类构造器的第一条语句。如果子类的构造器没有显示地调用超类的构造器，则将自动地调用超类默认（没有参数）的构造器。如果超类没有不带参数的勾走爱情，并且在子类的构造器中又没有显示地调用超类的其他构造器，则Java编译器将报告错误

方法覆盖

子类包含与父类同名方法的现象被称为方法重写，或方法覆盖。

方法的重写遵循：

  1. 方法名相同，形参列表相同
  2. 子类方法返回值类型应比父类方法返回值类型更小或相同
  3. 子类方法声明抛出的异常类应比父类方法声明抛出的异常类型更小或相等
  4. 子类方法的访问权限应比父类方法的访问权限更大或相等
  5. 覆盖方法和被覆盖方法要么都是类办法，要么都是实例方法。
  6. 如果父类方法具有private访问权限，则该方法对其子类是隐藏的，因此其子类无法访问该方法，也就是无法重写方法

super限定

  1. 如果需要在子类方法中调用父类中被覆盖的方法，则可以使用super或者父类类名作为调用者来调用父类中覆盖的方法。
  2. super不能出现在static修饰的方法中
  3. 如果在构造器中使用super，则super用于限定该构造器初始化的是该对象从父类继承得到的实例变量，而不是该类自己定义的实例变量
  4. 在一个构造器中调用另一个重载的构造器使用this调用来完成，在子类构造器中调用父类构造器使用super调用来完成。使用super调用父类构造器也必须出现在子类构造器执行体的第一行

多态

一个超类对象既可以引用一个超类对象，也可以引用超类的任何一个子类的对象。

因为子类其实是一种特殊的父类，因此Java允许把一个子类对象直接赋给一个父类引用变量，无须任何类型转换。

不能将一个超类的引用赋给子类变量

引用变量在编译阶段只能调用其编译时类型所具有的方法，但运行时执行它运行时类型所具有的方法。因此，在编译Java代码时，引用变量只能调用声明该变量时所用类里包含的方法。例如，通过Object p = new Person()代码定义的一个变量p，则这个p只能调用Object类的方法，而不能调用Person类定义的方法。

通过引用变量来访问其包含的实例变量时，系统总是试图访问它编译时类型所定义的成员变量，而不是它运行时类型所定义的成员变量。

引用变量的强制类型转换

编写Java程序时，引用变量只能调用它编译时类型的方法，而不能调用它运行时类型的方法，即使它实际所引用的对象确实包含该方法。如果需要让这个引用变量调用它运行时类型的方法，则必须把它强制类型转换成运行时类型。

引用类型之间的转换只能在就具有继承关系的两个类型之间进行，如果是两个没有任何继承关系的类型，则无法进行类型转换，否则编译时就会出现错误。

instanceof运算符：在进行强制类型转换之前，先判断一个对象是否是后一个类的实例，是否可以成功转换。instanceof运算符前面操作数的编译时类型要么与后面的类型相同，要么与后面的类具有父子继承关系，否则会引起编译错误。

动态绑定和静态绑定

  • 静态绑定：如果是private方法、static方法、final方法或者构造器，那么编译器将可以准确知道应该调用哪个方法，称为静态绑定
  • 动态绑定：调用的方法依赖于隐式参数的实际类型，称为动态绑定。

final类和方法

  • final类：用于阻止利用某个类定义子类，不允许扩展的类被称为final类
  • 类中特定的方法也可以被声明为final，子类不能覆盖这个方法
  • 如果将一个类声明为final，只有其中的方法自动地成为final，而不包括域

抽象类

  • 在某一个方法中使用abstract关键字，则不需要实现这个方法
  • 包含一个或多个抽象方法的类本身必须被声明为抽象的
  • 除了抽象方法，抽象类还可以包含具体数据和具体方法
  • 抽象方法充当着占位的角色，具体实现在子类中
  • 类即使不含抽象方法，也可以将类声明为抽象类
  • 抽象类不能被实例化
  • 可以定义一个抽象类的对象变量，但是它只能引用非抽象子类的对象
