C++函数

Posted on 2019-11-06 | In C++

函数

C++对于返回值的类型有一定限制，不能是数组，但可以是其他任何类型——整数、浮点数、指针，甚至可以是结构和对象

函数与数组

在C++中，当且仅当用于函数头或函数原型中，int *arr和int arr[]的含义相同

传递常规变量时，函数将使用改变量拷贝。传递数组时，函数将使用原来的数组。传递数组时，并没有将数组内容传递给函数，而是将数组的位置（地址）提交给函数

为将数组类型和元素数量告诉数组处理函数，请通过两个不同的参数来传递它们：void fillArray(int arr[]，int size)

用const保护数组

使用普通参数时，这种保护将自动实现，这是由于C++按值传递，并且函数使用数据的副本。然而接受数组名的函数将使用原始数据。为防止函数修改数组内容，可在声明形参时使用关键字const

void show_array(const double ar[],int n);

const和指针

  • 让指针指向一个常量对象，这样可以防止使用该指针来修改所指向的值

  • 将指针本身声明为常量，这样可以防止改变指针指向的位置

    尽可能将指针参数声明为指向常量数据的指针：

    • 这样可以避免由于无意间修改数据而导致的编程错误
    • 使用const使得函数能够处理const和非const实参，否则只能接受非const数据

函数与字符串

将字符串作为参数来传递，实际传递的是字符串第一个字符的地址

函数与结构

传递参数

  • 按值传递：函数将使用原始结构的副本

  • 引用传递：传递结构的地址，然后使用指针来访问结构的内容，函数使用的是原始结构

返回结构

函数指针

  • 获取函数的地址：只要使用参数名即可

  • 声明函数指针：声明应指定函数的返回类型以及函数的参数列表

引用变量

通过将引用变量用作参数，函数将使用原始数据而不是副本。

临时变量的生成

如果实参与引用参数不匹配，仅当参数为const引用时，C++将生成临时变量。

如果引用参数是const，则编译器将在下面两种情况生成临时变量：

  • 实参的类型正确，但不是左值
  • 实参的类型不正确，但可以转换为正确的类型

返回引用

应避免返回函数终止时不再存在的内存单元引用，为避免这种问题，解决方法有：

  • 返回一个作为参数传递给函数的引用

  • 用new来分配新的存储空间。这种方法存在一个问题，在不再需要new分配内存时，应使用delete来释放。

将引用用于类对象

默认参数

通过函数原型设置默认参数。只有原型指定了默认值，函数定义与没有默认参数时完全相同

对于带参数列表的函数，必须从右向左添加默认值。也就是说，要为某个参数设置默认值，则必须为它右边的所有参数提供默认值

实参按从左到右的顺序依次被赋给响应的形参，不能跳过任何参数
