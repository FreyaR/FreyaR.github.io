Spring Boot yaml

首先application.yaml在Spring Boot中可以写在四个不同的位置，分别是如下位置：

  1. 项目根目录下的config目录中
  2. 项目根目录下
  3. classpath下的config目录中（类路径下的，src/java/main/resources/config）
  4. classpath目录下（类路径下的，src/java/main/resources）

四个位置中的application.yaml文件的优先级按照上面列出的顺序依次降低。即如果有同一个属性在四个文件中都出现了，以优先级高的为准。

那么application.yaml是不是必须叫application.yaml这个名字呢？当然不是必须的。开发者可以自己定义yaml名字，自己定义的话，需要在项目启动时指定配置文件的名字

Environment variables: spring.config.name = app

当然这四个位置也不是一成不变的，也可以自己定义，有两种方式，一个是使用 spring.config.location属性，另一个则是使用 spring.config.additional-location这个属性，在第一个属性中，表示自己重新定义配置文件的位置，项目启动时就按照定义的位置去查找配置文件，这种定义方式会覆盖掉默认的四个位置，也可以使用第二种方式，第二种方式则表示在四个位置的基础上，再添加几个位置，新添加的位置的优先级大于原本的位置。
